
 데이터 딕셔너리
 
 자원을 효율적으로 관리하고 다양한 정보를 저장하는 시스템 테이블
 데이터 딕셔너리는 사용자가 테이블을 생성하거나 사용자를 변경하는 등의
 작업을 할 때 데이터베이스 서버에 의해 자동으로 갱신되는 테이블
 
 User_tables : 자신의 계정이 소유한 객체 등에 관한 정보를 조회 할 수 있는 딕셔너리
 
 
 ---------------------------------------------------------------------------
 
 
 DDL (DATA DEFINITION LANGUAGE) : 데이터 정의 언어
 
 객체를 생성, 수정, 삭제 등
 데이터의 전체 구조를 정의하는 언어 주로 DB관리자,설계자가 사용
 
 객체(OBJECT) , 생성(CREATE) , 수정(ALTER) , 삭제(DROP)

제약 조건(CONSTRAINTS)

사용자가 원하는 조건의 데이터만 유지하기 위해 특정 컬럼에 설정하는 제약
데이터 무결성 보장을 목적으로 함.

--> 무결성 보장 == 데이터의 신뢰도 확보

+ 입력 데이터에 문제가 없는지 자동으로 검사하는 목적
+ 데이터의 수정/삭제 가능여부 검사등을 목적으로 함 
  --> 제약조건을 위배하는 DML 구문은 수행할 수 없음!

제약조건 종류
PRIMARY KEY, NOT NULL, UNIQUE, CHECK, FOREIGN KEY.
